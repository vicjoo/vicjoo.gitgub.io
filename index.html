Windows下同时配置GitLab&GitHub

@toc

前言

同一机器上，有时需要同时使用GitLab、GitHub、Gitee。下面介绍下自己的配置过程。

参考链接
https://www.jianshu.com/p/68578d52470c
https://blog.csdn.net/flowerspring/article/details/104962002
https://www.cnblogs.com/hafiz/p/8146324.html

清除git全局变量设置

公司的笔记本一直只用了部门的一个GitLab,已经设置了全局的user.name和user.email为公司的邮箱名称，需要先取消掉。如果没有设置，可忽略此步骤。

相关git命令如下

# 查看
$ git config --global --list
# 设置  
$ git config --global user.name "zhuweiwei"
$ git config --global user.email "zhuweiwei@xxxsoft.com"
# 取消设置
$ git config --global --unset user.name
$ git config --global --unset user.email

生成SSH Keys

新建一个文件夹，存放生成的keys。如：D:\sshkeys。
也可以直接生成在~/.ssh文件夹中，Windows下表示C:\Users\用户名\.ssh\文件夹下。就免去了下面拷贝的步骤了。

GitLab

  1. 进入sshkeys文件夹
  2. 生成密钥：填写file名称，填写密码。这里直接回车不设置密码。

或者直接指定路径和文件名，使用这种方式更方便

直接3次回车，默认无密码。

  1. SSH keys设置
    以GitLab为例，进入gitlab-> User Settings -> SSH Keys;将id_rsa_gitlab.pub中的内容拷贝到key中，title中填入名称，点击Add key按钮，添加完毕。

GitHub

同上生成GitHub的密钥，然后添加都GitHub中。

ssh-keygen -t rsa -f id_rsa_github -C "vicjoo@163.com"

使用

配置config文件

将d:\sshkeys中的文件拷贝到~/.ssh/目录下，Windows系统为C:\Users\用户名\.ssh\下（_直接创建了该文件夹并将keys创建到此文件下的话忽略拷贝步骤_）。并在该目录下新建config文件，内容如下（windows下也使用~/.ssh/）：

# gitlab
    Host gitlab
    HostName 192.168.131.211
    PreferredAuthentications publickey
    IdentityFile ~/.ssh/id_rsa_gitlab
    User git
# github
    Host github.com
    HostName github.com
    PreferredAuthentications publickey
    IdentityFile ~/.ssh/id_rsa_githab
    User git
# gitee
    Host gitee.com
    HostName gitee.com
    PreferredAuthentications publickey
    IdentityFile ~/.ssh/id_rsa_gitee
    User git
  • Host myhost（这里是自定义的host简称，以后连接远程服务器就可以用命令ssh myhost)
  • User 登录用户名(如：git)
  • HostName 主机名可用ip也可以是域名(如:github.com或者bitbucket.org)
  • Port 服务器open-ssh端口（默认：22,默认时一般不写此行)
  • IdentityFile 证书文件路径（如~/.ssh/id_rsa_*)

测试

GitLab

添加ssh key 到 ssh-agent（看实际情况，非必须）

# 查看ssh-agent是否处于开启状态，打印pid表示启用中
$ eval "$(ssh-agent -s)"
Agent pid 247
# 添加
$ ssh-add ~/.ssh/id_rsa_gitlab

测试

$ ssh -T git@gitlab
Welcome to GitLab, @zhuweiwei!

下载工程

$ git clone git@gitlab:Z1ZB1811107A_CORE4/core4-leaf-si-center/sicp4-treatment/sicp4-treatment-center.git

在本地以后gitlab仓库，git pull ,git commit -m “XXX”,git push 会提示 tell me who you are

此时需要填写local 用户名和邮箱，而不是 –global

$ ssh git config --local user.name "你的名字"
$ ssh git config --local user.email  "你的邮箱"

GitHub、Gitee 与上面描述相似
